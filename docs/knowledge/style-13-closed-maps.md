# 13. Closed Maps (クローズドマップ - マップオブジェクト)

## 制約 (Constraints)
*   **クラス定義の禁止:**
    `class` や `constructor` などのクラス関連構文・キーワードを一切使用しません。
*   **マップによる表現:**
    オブジェクトは、データプロパティと関数（メソッド）をキーと値のペアで保持するプレーンなマップ（JavaScript/TypeScriptのプレーンオブジェクト）として表します。
*   **明示的な `self` の引き渡し:**
    マップ内のメソッドを実行する際は、暗黙的な `this` に依存せず、必ず第一引数にマップオブジェクト自身（`self`）を明示的に渡して状態へのアクセスや他のメソッド呼び出しを行います。

---

## コードの特徴・解説
今回の実装では、型定義（`type`）とファクトリ関数を用いてオブジェクトを作成し、関数内部で `self` を操作するように記述しています。

*   **プロトタイプライクなマップ定義:**
    ```typescript
    type ProductMap = {
      id: string;
      name: string;
      stock: number;
      // メソッドも関数の型として格納。第一引数に self を取る
      has_stock: (self: ProductMap, qty: number) => boolean;
      decrease_stock: (self: ProductMap, qty: number) => void;
    };
    ```
*   **明示的な self 渡しによる呼び出し:**
    処理の中で他のメソッドや自身のプロパティを操作する場合、常に引数の `self` 経由で行います。
    ```typescript
    const mouse = catalog.find(catalog, 'PRD-02');
    if (mouse) cart.add_item(cart, mouse, 1);
    ```

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**クラスシステムや言語機能としてのオブジェクト指向がなくても、連想配列（マップ）と関数ポインタさえあればオブジェクト指向を再現できる**という点にあります。
Pythonにおけるメソッド呼び出し時の `def method(self, ...)` や、Luaにおけるテーブルを使ったオブジェクト指向、JavaScript初期のプロトタイプによるオブジェクト構築は、まさにこのパターンで動いています。

### 難しかった点 / 気づき
*   **「閉じた（Closed）」マップの概念:**
    JavaScript のレキシカルクロージャを利用すれば、`self` を明示的に引き回さなくても「メソッド内に状態を閉じ込める（カプセル化する）」ことは容易です。しかし、このスタイルでは「クロージャによる状態の隠蔽」ではなく、「マップ自体に状態と関数をバインドし、引数で状態を渡す」という動的なアプローチに重きが置かれています。
*   **型定義の複雑さ:**
    TypeScriptにおいて再帰的なマップ定義（引数で自身の型を受け取る）を行う場合、インターフェースや型エイリアスを定義する必要があります。これにより、ある程度の静的チェックが働きますが、実行時にマップのキーを書き換えることで挙動を動的に変更できるという動的な柔軟性も併せ持っています。

### 実務への応用
*   **動的な挙動拡張:**
    プログラムの実行時に、特定のインスタンスにのみ新しいメソッドやデータを動的に注入して挙動を書き換える（モンキーパッチや動的アスペクト）といった設計に利用できます。
*   **非オブジェクト指向言語でのモデリング:**
    Go言語など、伝統的なクラスが存在しない言語において、構造体（Struct）とレシーバー関数によるカプセル化や、マップを用いたモック（Mock）の構築を行う際の基礎的な考え方になります。
