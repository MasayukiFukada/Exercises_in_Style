# 07. Code Golf (コードゴルフ)

## 制約 (Constraints)
*   **物理的な文字数の最小化:**
    プログラム全体のソースコード量を極限まで減らします。不要なスペースや改行、コメントは徹底的に省かれます。
*   **極端な短縮記法と変数名の使用:**
    変数名や関数名はすべて1文字（`c`, `t`, `d`, `a`, `o` など）に短縮されます。
*   **ビルトイン機能の限界利用:**
    配列の `map`, `reduce`, `find` や論理演算子のショートサーキット（`||`）、三項演算子（`? :`）など、文字数を削るための言語仕様トリックをフル活用します。

---

## コードの特徴・解説
文字数を削減するために、以下のようなテクニックを使用しています：

*   **型指定と変数定義の結合:**
    `let c:..., t:...` のように1つの `let` 宣言で複数のグローバル変数をまとめ、文字数を削減。
*   **指数表記:**
    価格 `100000` を `1e5` と表記することで3文字を削減。
*   **ショートサーキット評価によるデフォルト値割当:**
    `let k=t[i]||0;` により、カートにない商品の初期値 `0` を数文字で実現。
*   **三項演算子によるインライン判定:**
    割引計算 `let w=s>=3000?Math.round(s*0.1):0;` などの条件分岐を最小化。
*   **カンマ演算子による処理連結:**
    `reduce` 内部で `let x=...,q=...,l=...` とまとめて宣言し、改行や `let` を削る。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**言語の文法規則と制限の限界に挑む「コードのパズル（知的遊戯）」**です。
実務でこの文体を使うことは「悪夢」ですが、言語仕様を細部まで深く理解していなければ、最短の記述で同じロジックをバグなく表現することはできません。言語への深い習熟度を測る良いバロメーターになります。

### 難しかった点 / 気づき
*   **デバッグの困難さ:**
    変数名が `c`, `x`, `k`, `a` などであるため、デバッガを動かしたときに「どの変数が何を指しているのか」の判別が非常に困難になります。エラーメッセージの指定（テストストーリーの出力文字列との一致）を守りつつコードを圧縮する際、1文字のタイプミスが全体のロジック崩壊に直結し、普段いかに「意味のある命名（Meaningful Names）」に助けられているかを痛感しました。
*   **「書く楽しさ」と「読む苦痛」の非対称性:**
    いかに短く書くかという「コードゴルフ」は非常に楽しく没頭できますが、翌日自分でそのコードを読み直したときには、まるで他人が書いた暗号を解読するような強い苦痛を感じます。

### 実務への応用
*   **コードのミニファイ (Minification) の理解:**
    JavaScript/TypeScriptを本番環境（ブラウザ）へデプロイする際、Terser や esbuild などのツールがコードを圧縮（Minify）します。このスタイルはそのミニファイアの変換ロジックを手動で体験していると言え、ビルドツールの仕組みへの理解に繋がります。
*   **ワンライナーやシェルスクリプト:**
    CLIコマンドのパイプラインや、ちょっとした使い捨てのスクリプトをターミナル上で素早く書き上げる際に、この短縮トリックが役に立ちます。
