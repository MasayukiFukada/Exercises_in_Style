# 42. Point-free (ポイントフリー)

## 制約 (Constraints)
- アプリケーションロジックを記述する関数において、仮引数（変数のバインド）やローカル変数の宣言（`const`, `let`）を一切禁止する。
- アプリケーションコード内では、ラムダ式での引数定義（例: `x => x.price`）も極力排除する。
- すべての処理フローは、カリー化された汎用ユーティリティ（`pipe`, `ifElse`, `map` など）を合成することによって構築する。

## コードの特徴・解説
- 状態 `AppState` を入力とし、新しい `AppState` を返す関数のパイプラインとして全体のショッピングシミュレーションを構成しています。
- 通常は `if` 文やローカル変数で行う条件分岐やデータ抽出を、`ifElse`, `prop`, `propEq` などの論理・ユーティリティ関数を使ってポイントフリーに合成しています。

## 自分なりの消化・考察 (Personal Insights)
- **理解의 ポイント:** 変数に名前を付けないことで、コードが「どう処理するか（命令）」ではなく「データの流れがどう定義されているか（宣言）」に集中します。可読性がパズルのようになりがちですが、適切に分割された合成関数は非常に美しく、バグの入り込む余地を減らせます。
- **難しかった点 / 気づき:** JavaScript/TypeScriptで完全なポイントフリーを書くためには、カリー化と引数の適用タイミング（遅延評価）に注意する必要があります。特に、`always(console.log(...))` のような即時評価バグを防ぐため、`tap(log(...))` のような遅延可能な高階関数への配慮が不可欠でした。
- **実務への応用:** 関数型プログラミング（特にRxJSやRamda、Lodash/fpなど）を用いたデータ処理パイプラインの設計において、冗長なアロー関数を減らし、宣言的で凝縮度の高いコードを書くためにこの考え方が活かせます。
