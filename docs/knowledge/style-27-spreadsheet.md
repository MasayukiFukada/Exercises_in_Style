# 27. Spreadsheet (スプレッドシート - リアクティブ)

## 制約 (Constraints)
*   **セルの概念と数式:**
    すべてのデータは「セル」オブジェクトに格納します。セルは、生の値（Value）を保持するか、他のセルに依存する「数式（Formula）」を設定します。
*   **自動的な更新伝播 (Reactive Propagation):**
    あるセルの値が変更されたとき、そのセルに依存している他のすべてのセル（数式セル）が、手動の指示なしに自動的（暗黙的）に再計算されて値が更新されます。
*   **命令的フローの排除:**
    `recalculateSubtotal()` のような「明示的に再計算メソッドを呼ぶ」手続き型フローを排し、セル同士の関係性の宣言だけでシステムを動かします。

---

## コードの特徴・解説
今回の実装では、依存セルへのリスナー（オブザーバー）と動的再計算ロジックを持った `Cell` クラスを自作し、スプレッドシートの数式評価と更新伝播を再現しています。

*   **依存関係グラフの構築:**
    カートアイテムの数量が変わると小計が、小計が変わると全体の合計が、合計が変わると割引額と最終合計が芋づる式に更新されるよう、依存関係（`addDependency`）を事前に宣言します。
    ```typescript
    // 例: 割引額はカート合計金額に依存する
    const discount = new Cell<number>(() => {
      const sub = cartSubtotal.get();
      return sub >= 3000 ? Math.round(sub * 0.1) : 0;
    });
    discount.addDependency(cartSubtotal);
    ```
*   **セルのリセットによる自然な伝播:**
    チェックアウト後にカートの数量セル（`laptopQty` など）を `0` にリセットする（`set(0)`）だけで、余計なクリア処理を書かずに、支払合計セルや小計セルが自動的に `0` へとクリアされます。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**プログラムを「手順（手続き）」ではなく「データの関係（宣言）」として記述すること**です。
スプレッドシート（Excel）でセルA1を変えるとA2の数式結果が自動で変わるように、コードで状態更新の順序やタイミングを考慮する責任から開発者が解放されます。

### 難しかった点 / 気づき
*   **依存グラフの手動構築:**
    今回の簡易実装では `addDependency` を手動で呼び出して関係性を結びつけていますが、依存方向を誤ると無限ループ（循環参照）になったり、再計算漏れが発生します。現代のリアクティブフレームワークでは、ゲッターへのアクセスを自動で検知して依存関係を「自動的（マジック）」に解決する仕組みが取り入れられています。

### 実務への応用
*   **リアクティブなフロントエンドフレームワーク (React, Vue, SolidJS, Svelte):**
    Vue.js の `computed` プロパティや、SolidJS の `createMemo`、Svelte の `$:` 構文は、まさにこの Spreadsheet スタイルそのものです。状態（State）が変わると、依存する計算値（Derived State）やUIの DOM が自動で更新されます。
*   **RxJS などのストリームプログラミング:**
    リアクティブプログラミングを用いて、非同期のイベント（マウスクリックやWebSocketデータ）が流れてきた際、それに依存するデータや表示を順次パイプラインで自動更新する設計に直結します。
