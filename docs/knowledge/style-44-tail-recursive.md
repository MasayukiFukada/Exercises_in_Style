# 44. Tail-Recursive (末尾再帰・トランポリン)

## 制約 (Constraints)
- すべてのループ構文（`for`, `while` など）の使用を禁止する。
- 配列の反復処理メソッド（`map`, `filter`, `reduce`, `forEach`, `find` など）の使用も禁止する（内部でループが走るため）。
- すべての繰り返し処理は、末尾位置（Tail Position）での自己再帰または相互再帰のみで実装する。
- JS/TSでのコールスタック制限を回避するため、自作の「トランポリン (Trampoline)」エンジンを用いて処理を平坦化（bounce）して実行する。

## コードの特徴・解説
- 繰り返し処理は、次の再帰呼び出しをクロージャ（`() => nextStep(...)`）としてラップして返すことで、関数がいったん制御を戻します。
- `bounce` ヘルパーが、返されたクロージャをスタックを積まずにループで順次実行（平坦化）します。
- 商品検索（`findByIdRec`）やカタログ出力（`printCatalogLinesRec`）など、すべての反復処理がこの仕組みに適合する形で実装されています。

## 自分なりの消化・考察 (Personal Insights)
- **理解のポイント:** ループや状態の変更を禁止し、再帰と引数引き回しのみで繰り返しを定義することで、手続きではなく「状態の数学的な遷移」として反復をとらえることができます。
- **難しかった点 / 気づき:** JavaScript/TypeScriptは（一部環境を除いて）コンパイラレベルでの末尾再帰最適化（TCO）を自動で行わないため、これを手動でシミュレートするトランポリンの導入が必要です。関数の末尾で即時実行するのではなく、遅延実行用の関数を返すという思考の切り替えがパズル的でした。
- **実務への応用:** 計算幾何学や木構造・グラフ構造の深さ優先探索など、深い再帰構造を持つアルゴリズムにおいて、ランタイムのコールスタック制限を超えて安全に処理を実行するためのテクニックとして極めて重要です。
