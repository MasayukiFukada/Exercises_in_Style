# 37. Bowtie (ボウタイ - オートエンコーダ)

## 制約 (Constraints)
*   **ボトルネック層を持つ多層構造:**
    入力層の次元に対し、中間層（隠れ層）の次元を小さくし、出力層で再び元の次元へ戻す「ボウタイ（蝶ネクタイ）」状のネットワーク構造を定義します。
*   **エンコードとデコード:**
    入力情報を低次元へ射影する「エンコード（圧縮）」処理と、圧縮された特徴から元の情報を再構成する「デコード（復元）」処理の2段階で行列演算を行います。
*   **特徴量による意思決定:**
    元のデータを直接ロジックに渡すのではなく、一度このボトルネック層を通過して得られた抽象特徴量（または復元された情報）を利用してビジネスの判断を下します。

---

## コードの特徴・解説
今回の実装では、カート内の3つの商品の数量 `[ノートPC, マウス, キーボード]` を 3次元ベクトルとして入力し、1次元のボトルネック層（購買規模）へ圧縮し、再度 3次元へ復元する `BowtieNetwork`（簡易オートエンコーダ）を定義しています。

*   **エンコーダによる圧縮 (3D -> 1D):**
    各商品の価格比率に応じた重み $W_{enc}$ を用いて、3つの数量を「総合購買規模（小計に近い特徴量）」の 1次元に圧縮します。
*   **デコーダによる復元 (1D -> 3D):**
    1次元の購買規模から、各商品の数量パターンを再構成します。
    ```typescript
    class BowtieNetwork {
      // 3 -> 1 -> 3 の次元推移
      forward(x: number[]) {
        const h = relu(dot(this.W_enc, x) + this.b_enc); // 圧縮
        const y1 = relu(h * this.W_dec[0][0] + this.b_dec[0]); // 復元1
        const y2 = relu(h * this.W_dec[1][0] + this.b_dec[1]); // 復元2
        const y3 = relu(h * this.W_dec[2][0] + this.b_dec[2]); // 復元3
        return { bottleneck: h, output: [y1, y2, y3] };
      }
    }
    ```
*   **特徴量ベースの割引:**
    ボトルネック値 `bottleneck >= 3`（合計が3000円以上に相当する特徴量）をトリガーにして割引額の適用を決定しています。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**情報の本質的な「特徴量」を抽出し、ノイズを排除すること**です。
オートエンコーダ（Bowtie）は、入力データを意図的に「狭い通路（ボトルネック）」に通すことで、無駄な情報やノイズを削ぎ落とし、そのデータが持つ本質的なパターン（例: 「この購入は高額PC購入パターンなのか、小物の合わせ買いパターンなのか」など）だけを抽出します。

### 難しかった点 / 気づき
*   **オートエンコーダの役割のシミュレート:**
    カート内の生の数量 `[1, 1, 0]` をそのまま if 文で見るのではなく、一旦 1次元（103.00）という購入規模に落とした上で、そこから復元された数量予測（ノートPC 1.03台、マウス 33.99台）を得るという流れが、通常のビジネスロジックからはかなり回りくどく感じられます。しかし、これがレコメンデーションエンジンや、ユーザーのクラスタリング（分類）の背後で動いている数学的プロセスの縮小版であることが理解できます。

### 実務への応用
*   **画像や音声のノイズ除去 (Denoising Autoencoder):**
    ノイズの混ざった入力データを Bowtie ネットワークに通すことで、中間層で特徴だけが抽出され、デコードされた出力では余計なノイズが消えて綺麗な画像や音声が復元されます。
*   **推薦システム（協調フィルタリング）:**
    ユーザーの膨大な閲覧履歴やカート履歴（超高次元）をオートエンコーダで低次元（興味のカテゴリ）に圧縮し、そこから別のおすすめ商品ベクトルをデコード（復元・予測）して推薦リストを作成する手法に広く使われています。
