# 41. Convolutions (畳み込み)

## 制約 (Constraints)
- **汎用的な畳み込み関数の定義**: 特定のビジネスロジックに特化しない、入力配列、カーネル（重みフィルター）、およびストライド（スライド歩幅）を受け取る汎用的な1次元または多次元の畳み込み（`convolve`）関数を定義して使用する。
- **個別ロジックループの排他**: 商品価格や在庫数、カート内数量を個別にチェックして計算するような手続き型ループや `if-else` 分岐を排する。
- **カーネル設計へのマッピング**: 在庫余裕や小計計算、割引適用といったあらゆる判定や集計を、対応する「入力テンソル」と「重みフィルター（カーネル）」の組み合わせにマッピングし、畳み込み結果と活性化関数によって算出する。

## コードの特徴・解説
- 汎用的な1次元畳み込み関数 `convolve` を定義しています。これは入力配列に対してカーネルをスライドさせながら局所的な積和を求めていく、デジタル信号処理やCNN（畳み込みニューラルネットワーク）の標準的なアルゴリズムそのものです。
- **在庫チェック**: 入力 `[在庫_i, カート_i, 要求_i]` に対し、在庫判定カーネル `[1, -1, -1]` をストライド `3` で適用し、各商品の在庫の余り具合を一挙に算出。結果にステップ関数を適用して十分性を判定します。
- **小計の計算**: カート内数量配列 `[qty_0, qty_1, qty_2]` に対し、商品価格配列 `[price_0, price_1, price_2]` をカーネルとして指定し、カーネルサイズと同じストライドで畳み込むことで、一挙に合計金額を算出しています。
- **割引判定**: 小計配列に対してカーネル `[1]` を畳み込み、バイアス `-3000` を差し引いてステップ関数を通すことで、割引適用フラグを計算しています。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
*   **重み積和によるフィルタリング:** インデックス切り出しに焦点を当てた `Sliding Window` に対し、こちらは切り出したデータへのカーネル（重みフィルター）の適用とスライド積和（デジタル信号処理の畳み込みそのもの）に焦点を当てています。
*   **畳み込みの表現力:** カートの小計計算が「数量配列と価格配列（カーネル）の畳み込み」としてループなしで表現できることなど、数学的な表現力の高さを実感しました。

### 難しかった点 / 気づき
*   **入力とカーネルの二分化:** 本来ループ処理であるものを畳み込みに変換する際、「どのデータが入力配列で、どのパラメータがカーネル（重みフィルター）なのか」の概念的な割り当てを設計する部分に工夫が必要でした。
*   **ビジネスルールの再定義:** 畳み込みという高度な数値演算プリミティブだけを用いて、在庫管理や割引判定、売上計算といった業務ロジックのすべてを再定義できたのが非常に面白い経験でした。

### 実務への応用
*   **デジタル信号・時系列処理:** 音声や地震波などの波形データのノイズ除去（平滑化フィルター）や、センサーログの移動平均計算、画像データの輪郭抽出処理などに直接応用できます。
*   **並列データ解析:** 行列演算ライブラリを利用して膨大なトランザクションデータから特定パターンのイベント（例: 連続購入や同一元IPからの急増など）を高速に検出・集計するタスクに有効です。
