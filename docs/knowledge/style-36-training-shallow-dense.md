# 36. Training Shallow Dense (学習)

## 制約 (Constraints)
*   **固定パラメータの排除:**
    ニューラルネットワーク（パーセプトロン）の重み（$W$）やバイアス（$b$）の値を、コード内に固定値（ハードコード）としてあらかじめ記述することを禁止します。
*   **実行時学習 (Dynamic Training):**
    システム起動時、または処理開始前に、教師データ（入力と正しい判定結果のペア）をネットワークに注入し、重みとバイアスを動的に更新（学習）してパラメータを自動決定します。
*   **デルタ規則（パーセプトロン学習規則）の適用:**
    教師サンプルの入力に対する予測と正解との「誤差（Error）」を求め、その誤差に学習率と入力を掛け合わせて重みとバイアスを修正する更新ルールを適用します。

---

## コードの特徴・解説
今回の実装では、小計金額を受け取って割引適用を判定する `PerceptonModel` を用意し、起動時に `trainingSamples` から 500 エポックの学習を実行してパラメータを収束させています。

*   **入力の正規化スケーリング:**
    小計（数百〜数万）の値をそのまま重み計算に用いると、勾配（重み）が急激に爆発してしまい収束しないため、入力を「千円単位（`subtotal / 1000`）」に正規化スケーリングして学習させています。
*   **デルタ規則による学習ループ:**
    ```typescript
    train(trainingData: { x: number; y: number }[]): void {
      for (const sample of trainingData) {
        const xNorm = sample.x / 1000;
        const pred = this.predict(sample.x);
        const error = sample.y - pred; // 誤差
        if (error !== 0) {
          // パラメータの修正 (デルタ規則)
          this.w += this.learningRate * error * xNorm;
          this.b += this.learningRate * error;
        }
      }
    }
    ```
*   **学習結果の検証:**
    3000円以上で10%割引を適用するルールを正しく学習し、`w = 0.0440, b = -0.1300` に収束し、2500円（判定 0）と3500円（判定 1）の境界線を完璧に識別できています。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**ルールを人間が手で書くのではなく、データ（サンプル）からシステムに自己学習させるアプローチ**です。
通常「3000円以上は割引」という仕様が変わる（例えば「5000円以上」など）たびに、手続き型の if 条件文を修正する必要がありますが、機械学習アプローチでは、教師サンプルデータを変更して再学習させるだけで、プログラム自体を変更せずに新しいルールに適応できます。

### 難しかった点 / 気づき
*   **入力値のスケーリング（正規化）の重要性:**
    機械学習においてデータの正規化（Normalization）がいかに決定的な役割を持つかを、バグ（重みの爆発）を通じて痛感しました。値のスケールが異なるデータをそのままパーセプトロンに流すと学習が破綻するため、適切なスケーリングを施すことが必須となります。

### 実務への応用
*   **不正検知やレコメンドルールの動的学習:**
    ECショッピングにおいて、ユーザーの過去の行動データ（購入履歴や閲覧履歴）から、「どのようなカート状態のときに次の特定商品を推薦すべきか」の類似度・推薦ルールを裏側の学習プロセスで動的に更新し、それを本番推論に適用するシステムなどに適用されます。
