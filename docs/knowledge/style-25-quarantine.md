# 25. Quarantine (隔離 / 純粋関数)

## 制約 (Constraints)
*   **純粋関数と副作用の分離 (Impure/Pure separation):**
    副作用を一切持たない純粋な関数（同一入力に対し常に同一出力を返し、外部状態を書き換えない）と、画面出力（`console.log`）や状態の更新といった副作用（不純な処理）を厳密に隔離します。
*   **不純な処理のコンテナ化 (Quarantine):**
    不純な処理は、すべて「IOモナド（IOコンテナ）」の中にカプセル化して隔離します。
*   **遅延評価の強制:**
    プログラムの流れは、IOモナドを `bind` を使って結合するだけで構成し、最後の明示的な `run()` メソッドの呼び出しまで、いかなる副作用（出力や変更）も引き起こさせないようにします。

---

## コードの特徴・解説
今回の実装では、`IO` クラスを定義し、割引やレシートテキストの生成を純粋関数で行いつつ、画面出力と状態の更新を `IO` コンテナで隔離しています。

*   **純粋計算ロジックの抽出:**
    `getSubtotal`（小計計算）、`calculateDiscount`（割引計算）、`generateReceiptText`（レシート文字列生成）は、I/Oも外部状態の変更も行わない純粋関数です。
*   **IOコンテナへの隔離:**
    カタログの表示やカート状態の変更などの不純な処理は、`new IO(() => { ... })` によって隔離され、新しい状態を包んだ `IO` インスタンスを返します。
    ```typescript
    class IO<T> {
      private readonly effect: () => T;
      constructor(effect: () => T) { this.effect = effect; }
      bind<U>(fn: (val: T) => IO<U>): IO<U> {
        return new IO(() => fn(this.effect()).run());
      }
      run(): T { return this.effect(); }
    }
    ```
*   **遅延実行の構築:**
    ストーリーの流れは、IOモナドを組み立てていく形をとり、最後に `run()` を呼ぶことで順番に副作用が実行されます。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**プログラムの「状態の更新」や「実行順序の制御」を静的な値の結合（合成）として扱い、予測可能性を最大化すること**です。
通常の手続き型プログラミングでは、いつ画面出力が行われたり、いつ変数が書き換えられたりするかの順序依存（Temporal Coupling）が発生しますが、IOモナドに隔離することで、副作用の発生場所をコードの終端（`run()`）だけに「検疫（Quarantine）」できます。

### 難しかった点 / 気づき
*   **モナドのチェーンと状態管理:**
    `The One` スタイルでも同様にモナドを使いましたが、`The One` は「値を包み込んでメソッドチェーンで渡す」のが主目的でした。一方、`Quarantine` は「計算の実行（エフェクト）自体を関数として包み、実行を遅延させる」という点にあります。値を即座に変えるのではなく、実行を遅延した状態でパイプラインを構築する思考への切り替えが必要です。

### 実務への応用
*   **Haskell / 関数型プログラミング:**
    Haskellの `IO` アクションの設計思想そのものです。計算の実行順序が厳密に規定され、関数型の世界で安全に副作用を管理できます。
*   **TypeScriptにおけるFPライブラリ (fp-ts):**
    `fp-ts` の `IO` や `Task`（非同期処理を包むモナド）などを用いて、実務の複雑な非同期APIコールやDBの書き込みを副作用から隔離し、純粋なドメインロジックとして堅牢にテスト・実装する際に適用されます。
