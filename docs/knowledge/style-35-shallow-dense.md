# 35. Shallow Dense (浅い密なネットワーク)

## 制約 (Constraints)
*   **手続き的条件判断（if-else）の排除:**
    主要なビジネスの合否判断（このプロジェクトでは、カート追加の在庫判定や、精算時の割引対象判定）を、通常の `if-else` による命令的な比較式で記述することを排します。
*   **パーセプトロンによるフォワードパス:**
    入力データに対し、重み（Weights）との線形結合（ドット積）にバイアス（Bias）を加算し、活性化関数（ステップ関数など）を通すという、単層ニューラルネットワーク（フォワードプロパゲーション）の数式だけで結果（0 または 1）を出力します。
*   **固定されたパラメータ:**
    ネットワークが用いる重みやバイアスは、事前に定義された固定値（推論専用モデル）を使用します。

---

## コードの特徴・解説
今回の実装では、在庫判定と割引判定のビジネスルールを単層パーセプトロンに置き換えています。

*   **在庫判定パーセプトロン:**
    入力ベクトル $x = [\text{在庫}, \text{カート内}, \text{追加要求}]$ に対し、重み $W = [1, -1, -1]$ とバイアス $b = 0$ を設定。
    $W \cdot x + b = \text{在庫} - \text{カート内} - \text{追加要求}$ の計算結果が $0$ 以上なら $1$（追加可能）を返します。
    ```typescript
    function predictStockAvailability(stock: number, currentInCart: number, addQty: number): boolean {
      const x = [stock, currentInCart, addQty];
      const w = [1, -1, -1];
      const b = 0;
      const z = dotProduct(w, x) + b;
      return stepFunction(z) === 1;
    }
    ```
*   **割引判定パーセプトロン:**
    入力 $x = [\text{小計}]$ に対し、重み $W = [1]$ とバイアス $b = -3000$ を設定。
    $W \cdot x + b = \text{小計} - 3000$ が $0$ 以上なら適用フラグ $1$（10%割引）を、負なら $0$（割引なし）を返します。

## 自分なりの消化・考察 (Personal Insights)

### 理解のポイント
このスタイルの本質は、**論理的なルール判断を数値計算（線形空間の分離）に変換すること**です。
通常「AはB以上か？」という if 分岐で書く処理も、数学的には「境界値（バイアス）で引いて、正か負かを活性化関数で判定する」という行列のフォワードパス計算と全く同じ構造になります。これにより、論理判定コードがシンプルな数値演算に統一されます。

### 難しかった点 / 気づき
*   **活性化関数の選定:**
    出力が確率（0〜1の実数）ではなく「適用か不適用か」の二値であるため、通常のシグモイド関数（Sigmoid）よりも、閾値を境界に 0 か 1 を明確に返すステップ関数（Heaviside step function）が論理判定の再現には最適です。

### 実務への応用
*   **ディープラーニングモデルの推論 (Inference):**
    学習済みのAIモデルを本番環境（WebサービスやIoTデバイス）にデプロイし、入力されたセンサーデータやユーザーデータから、フォワードプロパゲーション演算だけで高速に予測（不正検知、スパム判定など）を行う際の基本構成です。
